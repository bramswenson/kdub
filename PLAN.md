# yk-gpg — Development Plan

## Project Overview

Cross-platform GPG key lifecycle management with OpenPGP smart card support (YubiKey and others). Based on drduh/YubiKey-Guide, wrapped in mise tasks. Runs on Linux, macOS, and Tails OS.

**Primary target**: Tails 7.5+ (Debian Trixie, GnuPG 2.4.7)
**Also supported**: Ubuntu 24.04+ (GnuPG 2.4.4), macOS Homebrew (GnuPG 2.4.9)
**Minimum GnuPG**: 2.4+ (required for `gpg-card` CLI)

## Rename

Rename from `tails-yubikey-gpg-manager` to `yk-gpg`. The git remote/repo name should also change. All internal references update accordingly.

## Architecture

### Design Principles

1. **XDG-standard paths everywhere** — no hardcoded platform paths in core logic
2. **Platform detection with thin integration layers** — Tails, macOS, Linux auto-detected; special needs handled by dedicated tasks
3. **`gpg-card` as primary card interface** — portable, scriptable, no `ykman` dependency for standard operations
4. **`ykman` only for YubiKey-specific features** — touch policy is the only operation that requires it
5. **Batch mode for CI** — every interactive prompt has a non-interactive path via flags/env vars
6. **Security-first ephemeral GNUPGHOME** — tiered RAM-backed temp dirs, never touches disk when possible

### Directory Layout (XDG)

All platforms use identical logical paths:

```
~/.config/yk-gpg/                        # $YK_GPG_CONFIG_DIR
├── gpg.conf                             # hardened GPG config (from config/gpg.conf)
├── gpg-agent.conf                       # platform-specific (generated by setup:init)
├── scdaemon.conf                        # platform-specific (generated by setup:init)
└── dirmngr.conf                         # only if Tor proxy configured

~/.local/share/yk-gpg/                   # $YK_GPG_DATA_DIR
├── identities/
│   └── <fingerprint>.json               # per-identity metadata
└── backups/
    └── <fingerprint>/
        ├── certify-key.asc              # armored secret certify key
        ├── subkeys.asc                  # armored secret subkeys
        ├── public-key.asc              # armored public key
        ├── ownertrust.txt              # trust database export
        └── revocation-cert.asc         # revocation certificate

~/.local/state/yk-gpg/                   # $YK_GPG_STATE_DIR (not used on Tails)
└── logs/                                # operation logs (future)

$XDG_RUNTIME_DIR/yk-gpg/gnupg-XXXXXX/   # ephemeral GNUPGHOME (per-operation)
```

### Identity Metadata JSON Schema

Each identity gets `$YK_GPG_DATA_DIR/identities/<fingerprint>.json`:

```json
{
  "identity": "Name <email>",
  "fingerprint": "ABCDEF1234567890ABCDEF1234567890ABCDEF12",
  "keyid": "0x1234567890ABCDEF",
  "key_type": "ed25519",
  "expiration": "2y",
  "created": "2026-03-09T12:00:00Z",
  "backed_up": "2026-03-09T12:05:00Z",
  "renewed": "",
  "rotated": "",
  "card_serial": "",
  "card_model": "",
  "provisioned": ""
}
```

Fields are updated incrementally by each task via `save_metadata()`. Empty string means not yet set.

### Mise Configuration (Dual mise.toml)

The repo has **two** `mise.toml` files with different purposes:

1. **Root `mise.toml`** — development/CI: dev tools, test/lint tasks, shellcheck. This is what devs and CI use.
2. **`src/mise.toml`** — distribution: user-facing config with `[env]` defaults and the actual `gpg:*`, `card:*`, etc. tasks. This is what end users work from.

Mise discovers file tasks relative to the `mise.toml` that defines them. The root `mise.toml` discovers tasks in `mise-tasks/` (dev tasks). The `src/mise.toml` discovers tasks in `src/mise-tasks/` (product tasks).

**Root `mise.toml` (dev/CI — committed):**
```toml
[settings]
experimental = true
env_shell_expand = true

[env]
YK_GPG_SRC_DIR = "{{ config_root }}/src"
```

Dev tasks live in `mise-tasks/` at the repo root (e.g., `mise-tasks/test`, `mise-tasks/lint`).

**`src/mise.toml` (distribution — committed):**
```toml
[settings]
experimental = true
env_shell_expand = true

[env]
YK_GPG_KEY_TYPE = "ed25519"
YK_GPG_EXPIRATION = "2y"
YK_GPG_IDENTITY = ""
YK_GPG_TOR_PROXY = ""
YK_GPG_CONFIG_DIR = "${XDG_CONFIG_HOME:-$HOME/.config}/yk-gpg"
YK_GPG_DATA_DIR = "${XDG_DATA_HOME:-$HOME/.local/share}/yk-gpg"
YK_GPG_STATE_DIR = "${XDG_STATE_HOME:-$HOME/.local/state}/yk-gpg"
```

Product tasks are auto-discovered from `src/mise-tasks/`.

**`src/mise.local.toml.example` (committed as template, users copy to `src/mise.local.toml`):**
```toml
# Copy this file to mise.local.toml and customize.
# mise.local.toml is gitignored and overrides mise.toml values.
[env]
# YK_GPG_IDENTITY = "Your Name <you@example.com>"
# YK_GPG_KEY_TYPE = "rsa4096"
# YK_GPG_EXPIRATION = "1y"
# YK_GPG_TOR_PROXY = "socks5h://127.0.0.1:9050"
```

**`.gitignore` additions:**
```
mise.local.toml
```

**User workflow**: `cd src && mise run gpg:create -- "Name <email>"`
**Dev workflow**: `mise run test` (from repo root), or `cd src && mise run gpg:create` to test product tasks.

**Precedence** (highest wins): CLI flag → `src/mise.local.toml` → `src/mise.toml`

## Detailed Specifications

### `src/lib/platform.sh`

New file. Sourced by `src/lib/common.sh`. Contains all platform-specific logic.

```bash
#!/usr/bin/env bash
# Platform detection, XDG paths, and ephemeral GNUPGHOME management

# --- Platform detection ---

# Returns: "tails", "macos", or "linux"
detect_platform() {
    if [[ "$(uname)" == "Darwin" ]]; then
        echo "macos"
    elif [[ -f /etc/os-release ]] && grep -q "Tails" /etc/os-release 2>/dev/null; then
        echo "tails"
    else
        echo "linux"
    fi
}

PLATFORM="$(detect_platform)"

is_tails() { [[ "$PLATFORM" == "tails" ]]; }
is_macos() { [[ "$PLATFORM" == "macos" ]]; }
is_linux() { [[ "$PLATFORM" == "linux" || "$PLATFORM" == "tails" ]]; }

# --- Batch mode ---

BATCH_MODE="${BATCH_MODE:-${CI:+true}}"
is_batch() { [[ "${BATCH_MODE:-}" == "true" ]]; }

# --- XDG path resolution ---
# These use env vars set by mise.toml [env] section.
# If sourced outside mise (e.g., tests), fall back to XDG defaults.

yk_gpg_config_dir() { echo "${YK_GPG_CONFIG_DIR:-${XDG_CONFIG_HOME:-$HOME/.config}/yk-gpg}"; }
yk_gpg_data_dir()   { echo "${YK_GPG_DATA_DIR:-${XDG_DATA_HOME:-$HOME/.local/share}/yk-gpg}"; }
yk_gpg_state_dir()  { echo "${YK_GPG_STATE_DIR:-${XDG_STATE_HOME:-$HOME/.local/state}/yk-gpg}"; }

# --- Ephemeral GNUPGHOME ---

# Creates a RAM-backed temporary GNUPGHOME directory.
# Sets and exports: GNUPGHOME, _YK_GPG_TMPDIR_METHOD ("xdg_runtime"|"devshm"|"ramdisk"|"tmpfs"|"tmpdir")
# On macOS, also sets _YK_GPG_RAMDISK_DEVICE if hdiutil was used.
# Copies gpg.conf from project config/ and generates platform-specific
# gpg-agent.conf and scdaemon.conf into the new GNUPGHOME.
#
# Security tier fallback order:
# 1. Linux: $XDG_RUNTIME_DIR (already tmpfs, mode 0700, no sudo)
# 2. Linux: /dev/shm private subdir (tmpfs, mode 0700, no sudo)
# 3. macOS: hdiutil RAM disk (16MB, no sudo)
# 4. Linux: mount -t tmpfs (requires sudo)
# 5. macOS: $TMPDIR (per-user, mode 0700, disk-backed — warns)
# 6. Any: mktemp -d /tmp/... (checks if /tmp is tmpfs — warns if not)
setup_temp_gnupghome() { ... }

# Cleans up the ephemeral GNUPGHOME created by setup_temp_gnupghome().
# Kills gpg-agent, unmounts ramdisk/tmpfs if applicable, removes directory.
cleanup_temp_gnupghome() { ... }

# --- Networking ---

# Configures Tor proxy if YK_GPG_TOR_PROXY is set.
# Exports ALL_PROXY. Installs dirmngr.conf into the current GNUPGHOME.
setup_networking() {
    if [[ -n "${YK_GPG_TOR_PROXY:-}" ]]; then
        export ALL_PROXY="$YK_GPG_TOR_PROXY"
        # Install dirmngr.conf with proxy into current GNUPGHOME
        local gnupghome="${GNUPGHOME:-$(yk_gpg_config_dir)}"
        local project_root="${MISE_PROJECT_ROOT:-$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)}"
        if [[ -f "${project_root}/config/dirmngr.conf" ]]; then
            cp "${project_root}/config/dirmngr.conf" "${gnupghome}/dirmngr.conf"
        fi
    fi
}

# --- Dependency checking ---

# Checks that gpg and gpg-card are available. Does NOT check ykman (that's yubikey-only).
# On Linux, checks pcscd is running (starts it if possible).
# On macOS, pcscd is built-in (CryptoTokenKit) — no check needed.
check_gpg_deps() {
    local missing=()
    for cmd in gpg gpg-card; do
        command -v "$cmd" &>/dev/null || missing+=("$cmd")
    done
    # scdaemon: check both PATH and GnuPG libexec
    if ! command -v scdaemon &>/dev/null && [[ ! -x /usr/lib/gnupg/scdaemon ]] && [[ ! -x /usr/local/lib/gnupg/scdaemon ]] && [[ ! -x /opt/homebrew/lib/gnupg/scdaemon ]]; then
        missing+=("scdaemon")
    fi
    # pcscd: Linux only
    if is_linux && ! pgrep -x pcscd &>/dev/null; then
        if command -v systemctl &>/dev/null; then
            sudo systemctl start pcscd 2>/dev/null || missing+=("pcscd")
        elif command -v pcscd &>/dev/null; then
            sudo pcscd 2>/dev/null || missing+=("pcscd")
        else
            missing+=("pcscd")
        fi
    fi
    if [[ ${#missing[@]} -gt 0 ]]; then
        error "Missing dependencies: ${missing[*]}"
        error "Run: mise run setup:install-deps"
        return 1
    fi
}

# Checks that ykman is available. Only called by yubikey:* tasks.
check_ykman() {
    if ! command -v ykman &>/dev/null; then
        error "ykman (yubikey-manager) is not installed."
        error "Run: mise run setup:install-deps"
        return 1
    fi
}

# --- Config file generation ---

# Generates platform-specific gpg-agent.conf into the given directory.
# $1: target directory (GNUPGHOME or config dir)
generate_gpg_agent_conf() {
    local target_dir="$1"
    local pinentry_program
    case "$PLATFORM" in
        tails)  pinentry_program="/usr/bin/pinentry-gnome3" ;;
        macos)
            if [[ -x /opt/homebrew/bin/pinentry-mac ]]; then
                pinentry_program="/opt/homebrew/bin/pinentry-mac"
            elif [[ -x /usr/local/bin/pinentry-mac ]]; then
                pinentry_program="/usr/local/bin/pinentry-mac"
            else
                pinentry_program="/usr/bin/pinentry-curses"
            fi
            ;;
        *)      pinentry_program="/usr/bin/pinentry-curses" ;;
    esac
    cat > "${target_dir}/gpg-agent.conf" <<EOF
enable-ssh-support
pinentry-program ${pinentry_program}
default-cache-ttl 60
max-cache-ttl 120
allow-loopback-pinentry
EOF
}

# Generates platform-specific scdaemon.conf into the given directory.
generate_scdaemon_conf() {
    local target_dir="$1"
    if is_macos; then
        cat > "${target_dir}/scdaemon.conf" <<EOF
disable-ccid
pcsc-driver /System/Library/Frameworks/PCSC.framework/PCSC
EOF
    else
        cat > "${target_dir}/scdaemon.conf" <<EOF
disable-ccid
EOF
    fi
}
```

#### `gpg:restore` (new)

**File**: `src/mise-tasks/gpg/restore`
**Purpose**: Import keys from a backup directory into the current GNUPGHOME (ephemeral or persistent). This replaces the "import into persistent GNUPGHOME" logic that was in the old `gpg:backup`.

```bash
#!/usr/bin/env bash
#MISE description="Restore GPG keys from backup into current GNUPGHOME"
#USAGE arg "fingerprint" help="Key fingerprint to restore (matches backup dir name)"
set -euo pipefail

source "$MISE_PROJECT_ROOT/lib/common.sh"

FINGERPRINT="${usage_fingerprint:?Fingerprint argument required}"
BACKUP_DIR="$(yk_gpg_data_dir)/backups/${FINGERPRINT}"

if [[ ! -d "$BACKUP_DIR" ]]; then
    error "Backup not found: $BACKUP_DIR"
    exit 1
fi

info "Restoring from: $BACKUP_DIR"
gpg --import "${BACKUP_DIR}/public-key.asc"
gpg --import "${BACKUP_DIR}/certify-key.asc"
[[ -f "${BACKUP_DIR}/ownertrust.txt" ]] && gpg --import-ownertrust "${BACKUP_DIR}/ownertrust.txt"
success "Keys restored into GNUPGHOME=${GNUPGHOME:-$HOME/.gnupg}"
```

#### `card:setup`

**File**: `src/mise-tasks/card/setup`
**Purpose**: Configure PINs, KDF, cardholder metadata on any OpenPGP card via `gpg-card`.

```bash
#!/usr/bin/env bash
#MISE description="Configure OpenPGP card PINs, KDF, and cardholder metadata"
#USAGE flag "--identity <id>" help="GPG identity for name/login fields on card"
#USAGE flag "--url <url>" help="Public key URL to store on card"
#USAGE flag "--admin-pin <pin>" help="Current admin PIN (default: 12345678 for factory state)"
#USAGE flag "--new-admin-pin <pin>" help="New admin PIN (generated if omitted)"
#USAGE flag "--new-user-pin <pin>" help="New user PIN (generated if omitted)"
#USAGE flag "--skip-kdf" help="Skip KDF setup"
#USAGE flag "--factory-pins" help="Card has factory default PINs (skip prompt)"
set -euo pipefail

source "$MISE_PROJECT_ROOT/lib/common.sh"

check_gpg_deps

IDENTITY="${usage_identity:-}"
URL="${usage_url:-}"
SKIP_KDF="${usage_skip_kdf:-false}"
FACTORY_PINS="${usage_factory_pins:-false}"
ADMIN_PIN="${usage_admin_pin:-}"
NEW_ADMIN_PIN="${usage_new_admin_pin:-}"
NEW_USER_PIN="${usage_new_user_pin:-}"

# Determine PIN state
if is_batch; then
    # In batch mode, --factory-pins or --admin-pin is required
    if [[ "$FACTORY_PINS" == "true" ]]; then
        ADMIN_PIN="12345678"
    elif [[ -z "$ADMIN_PIN" ]]; then
        error "In batch mode, --factory-pins or --admin-pin is required"
        exit 1
    fi
else
    # Interactive: ask about PIN state
    if [[ -z "$ADMIN_PIN" ]]; then
        if confirm "Are PINs at factory defaults (Admin: 12345678, User: 123456)?" "y"; then
            ADMIN_PIN="12345678"
            FACTORY_PINS="true"
        else
            read -r -s -p "Enter current Admin PIN: " ADMIN_PIN
            echo ""
        fi
    fi
fi

# KDF setup (must happen while PINs are at factory defaults)
if [[ "$SKIP_KDF" != "true" && "$FACTORY_PINS" == "true" ]]; then
    info "Enabling KDF (PIN hashing)..."
    gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
kdf-setup
12345678
q
EOF
    success "KDF enabled"
fi

# PIN changes (only if factory defaults — otherwise user already changed them)
if [[ "$FACTORY_PINS" == "true" ]]; then
    NEW_ADMIN_PIN="${NEW_ADMIN_PIN:-$(tr -dc '0-9' < /dev/urandom | head -c 8)}"
    NEW_USER_PIN="${NEW_USER_PIN:-$(tr -dc '0-9' < /dev/urandom | head -c 6)}"

    if ! is_batch; then
        echo ""
        echo "=============================================="
        echo "  CARD PINs - RECORD THESE NOW"
        echo "=============================================="
        echo "  Admin PIN: $NEW_ADMIN_PIN"
        echo "  User PIN:  $NEW_USER_PIN"
        echo "=============================================="
        echo ""
        confirm "Have you recorded both PINs?" || { error "Aborting."; exit 1; }
    else
        info "New Admin PIN: $NEW_ADMIN_PIN"
        info "New User PIN:  $NEW_USER_PIN"
    fi

    # Change Admin PIN
    info "Changing Admin PIN..."
    gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
passwd
3
12345678
${NEW_ADMIN_PIN}
${NEW_ADMIN_PIN}
q
q
EOF
    ADMIN_PIN="$NEW_ADMIN_PIN"
    success "Admin PIN changed"

    # Change User PIN
    info "Changing User PIN..."
    gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
passwd
1
123456
${NEW_USER_PIN}
${NEW_USER_PIN}
q
q
EOF
    success "User PIN changed"
fi

# Set cardholder name and login
if [[ -n "$IDENTITY" ]]; then
    # Extract name from GPG key UID
    UID_STRING="$(gpg --with-colons --list-keys "$IDENTITY" 2>/dev/null | grep '^uid:' | head -1 | cut -d: -f10)"
    if [[ -n "$UID_STRING" ]]; then
        CARD_NAME="$(echo "$UID_STRING" | sed 's/ *<.*//')"
        CARD_LAST="$(echo "$CARD_NAME" | awk '{print $NF}')"
        CARD_FIRST="$(echo "$CARD_NAME" | sed "s/ *${CARD_LAST}\$//")"
        EMAIL="$(echo "$UID_STRING" | sed 's/.*<\(.*\)>.*/\1/')"

        info "Setting cardholder: $CARD_FIRST $CARD_LAST"
        gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
name
${CARD_LAST}
${CARD_FIRST}
${ADMIN_PIN}
q
EOF
        if [[ -n "$EMAIL" && "$EMAIL" != "$UID_STRING" ]]; then
            info "Setting login: $EMAIL"
            gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
login
${EMAIL}
${ADMIN_PIN}
quit
EOF
        fi
    fi
fi

# Set URL
if [[ -n "$URL" ]]; then
    info "Setting public key URL: $URL"
    gpg --command-fd=0 --pinentry-mode loopback --edit-card <<EOF
admin
url
${URL}
${ADMIN_PIN}
quit
EOF
fi

# Verify
echo ""
gpg --card-status
success "Card setup complete"
```

#### `card:provision`

**File**: `src/mise-tasks/card/provision`
**Purpose**: Transfer subkeys to OpenPGP card. Rejects factory-default PINs.

```bash
#!/usr/bin/env bash
#MISE description="Transfer GPG subkeys to OpenPGP smart card"
#USAGE arg "keyid" help="Key ID or fingerprint to provision"
#USAGE flag "--admin-pin <pin>" help="Card admin PIN (prompted if omitted)"
#USAGE flag "--passphrase <pass>" help="Certify key passphrase (prompted if omitted)"
#USAGE flag "--force" help="Skip confirmation prompts"
set -euo pipefail

source "$MISE_PROJECT_ROOT/lib/common.sh"

check_gpg_deps

KEYID="${usage_keyid:?Key ID argument required}"
ADMIN_PIN="${usage_admin_pin:-}"
CERTIFY_PASS="${usage_passphrase:-}"
FORCE="${usage_force:-false}"

KEYFP="$(gpg --with-colons --list-keys "$KEYID" 2>/dev/null | grep '^fpr:' | head -1 | cut -d: -f10)"
[[ -z "$KEYFP" ]] && { error "Key not found: $KEYID"; exit 1; }

info "Provisioning key $KEYID to card"

# Reject factory-default PINs
if is_batch; then
    [[ -z "$ADMIN_PIN" ]] && { error "--admin-pin required in batch mode"; exit 1; }
    [[ -z "$CERTIFY_PASS" ]] && { error "--passphrase required in batch mode"; exit 1; }
else
    if confirm "Are PINs at factory defaults?" "y"; then
        error "Run 'mise run card:setup' first to configure PINs."
        exit 1
    fi
    [[ -z "$ADMIN_PIN" ]] && { read -r -s -p "Enter card Admin PIN: " ADMIN_PIN; echo ""; }
    [[ -z "$CERTIFY_PASS" ]] && { read -r -s -p "Enter certify key passphrase: " CERTIFY_PASS; echo ""; }
fi

if [[ "$FORCE" != "true" ]] && ! is_batch; then
    warn "Subkeys will be MOVED to the card (local copy becomes a stub)."
    confirm "Transfer subkeys to card?" || { error "Aborted."; exit 1; }
fi

# Transfer: subkey 1 (sign) → slot 1, subkey 2 (encrypt) → slot 2, subkey 3 (auth) → slot 3
info "Transferring subkeys..."
gpg --command-fd=0 --pinentry-mode loopback --edit-key "$KEYFP" <<EOF
key 1
keytocard
1
${CERTIFY_PASS}
${ADMIN_PIN}
key 1
key 2
keytocard
2
${CERTIFY_PASS}
${ADMIN_PIN}
key 2
key 3
keytocard
3
${CERTIFY_PASS}
${ADMIN_PIN}
save
EOF

success "Subkeys transferred to card"

# Verify
info "Verification (keys should show 'ssb>' stubs):"
gpg -K "$KEYFP"
gpg --card-status

# Save metadata
CARD_SERIAL="$(gpg --card-status 2>/dev/null | grep 'Serial number' | awk '{print $NF}')"
save_metadata "$KEYFP" \
    "card_serial" "${CARD_SERIAL:-unknown}" \
    "provisioned" "$(date -u +%Y-%m-%dT%H:%M:%SZ)"

success "Card provisioning complete"
```

## Platform Knowledge Reference

### Tails OS

- **Tor SOCKS proxy**: `127.0.0.1:9050`. Set `YK_GPG_TOR_PROXY=socks5h://127.0.0.1:9050` in `mise.local.toml`.
- **`http_proxy`/`https_proxy`**: Do NOT set to `socks5h://`. Use `ALL_PROXY` for SOCKS.
- **GPG dirmngr**: Ignores env vars. Needs `dirmngr.conf` with `http-proxy socks5h://127.0.0.1:9050`.
- **apt**: Routed through Tor by Tails firewall rules. No env vars needed.
- **sudo**: Disabled by default. Must set Administration Password at Welcome Screen before boot.
- **Persistent storage**: `/home/amnesia/Persistent/` is bind-mounted. Dotfiles at `/live/persistence/TailsData_unlocked/dotfiles/` are symlinked to `~/`.
- **scdaemon/pcscd conflict**: `scdaemon.conf` needs `disable-ccid`.

### macOS

- **GnuPG**: `brew install gnupg pinentry-mac`
- **YubiKey Manager**: `brew install ykman`
- **No pcscd**: macOS uses built-in CryptoTokenKit/PCSC.framework
- **scdaemon**: Needs `disable-ccid` + `pcsc-driver /System/Library/Frameworks/PCSC.framework/PCSC`
- **`ykman` vs `scdaemon`**: Both want exclusive card access. May need `gpgconf --kill scdaemon` before `ykman`.
- **pinentry**: Use `pinentry-mac` (Homebrew). Apple Silicon: `/opt/homebrew/bin/pinentry-mac`. Intel: `/usr/local/bin/pinentry-mac`.
